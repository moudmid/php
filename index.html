Exercice 2

Vous avez trouvé un job d'été de manutantionnaire dans une grande surface. Votre travail consiste à ériger des pyramides de boîtes en tête de rayon. Ces boîtes sont conditionnées dans des cartons.

Votre responsable vous demande très régulièrement s'il est possible ou non de construire une pyramide en utilisant exactement toutes les boîtes contenues dans un carton. Pour qu'une pyramide soit constructible, son sommet doit se terminer par une et une seule boîte, en sachant que chaque étage possède 2 boîtes de moins que l'étage inférieur. Si la pyramide est constructible, votre responsable veut également savoir, pour des raisons d'encombrement, le nombre de boîtes qui en constitueront la base.

Vous décidez alors d'écrire en php un petit formulaire qui répond à cette question. De plus, ce petit outil dessine la pyramide lorsque la réponse est positive.

Par exemple, si vous posez la question pour un carton contenant 16 boîtes, la réponse sera :

Il est possible de construire une pyramide avec 16 boîtes.
La base sera constituée de 7 boîtes :
X
X X X
X X X X X
X X X X X X X

Exercice 3

Le but de cet exercice est de mettre en oeuvre un procédé de dissimulation d'information ou de chiffrement. Il s'agit de faire subir à un message (une chaîne de caractères) une transformation afin que la chaîne produite n'ait aucune signification pour qui viendrait à la lire (de façon légale ou illégale).
Peut-être connaissez-vous la transformation appelée 'chiffrement affine' ? Il consiste à remplacer chaque lettre du message par sa position p dans un alphabet (en partant de la position 0), d'appliquer à ce nombre une transformation affine du type ap+b, d'en prendre le reste dans la division par le nombre d'éléments de l'alphabet, puis de renvoyer la lettre de l'alphabet correspondant à cette nouvelle position.
Nous ferons le choix d'un alphabet à 73 éléments, à savoir et dans cet ordre, les 26 lettres minuscules, les 26 lettres majuscules, les 10 chiffres (dans l'ordre croissant) ainsi que les 9 signes de ponctuation : « » (l'espace), « . », « ? », « ! », « , », « ; », « : », « ' », « - », « ( » et « ) ».

Exemple pour a=3 et b=5 : la lettre N sera transformée en X.
En effet, la lettre N est au rang 39 dans l'alphabet, 3*39+5=122, 49 est le reste de la division de 122 par 72 et X est au rang 49 dans l'alphabet.

Créer une fonction chiffre_affine ayant pour entrée les éléments : a, b et chaine, et qui renvoie la chaîne de caractère correspondant à la transformation de la chaîne chaine par le chiffrement affine de paramètres a et b.
Parmi les exemples d'utilisation de votre outil, quel est le résultat de la transformation du texte :

lHcV2ScFZ2n02Z5xcnZ5cZkcnpxZjeZk2Faxc0ZjJjwZp2FNZ5cNZN255cNZwJEZcAZwJEPxNM

pour les valeurs a=28 et b=21 ?
Créer dans le présent fichier exercice_3.php un formulaire qui permette à l'internaute de choisir les trois éléments a, b, chaine et qui renvoie le résultat de votre outil chiffre_affine sur ces entrées.

Vous allez dans ce qui suit mettre en place et étudier une base de données.

Tout d'abord, créez dans phpmyadmin un nouvel utilisateur examen2021 dont le mot de passe sera également examen2021, ainsi qu'une base de données portant son nom.

Les informations sur les tables sont dans deux fichiers :

  • les structures dans examen2021_structures.sql qu'il est possible d'importer directement avec phpmyadmin ;
  • les données dans examen2021_donnees.sql qu'il n'est, pour des considérations de taille de fichier, pas possible d'importer de la même façon, il faudra suivre les instructions suivantes après avoir importé le fichier des structures :
    • déposer le fichier dans le répertoire volume_mariadb de votre machine virtuelle,
    • pénétrer dans le container mariadb :
      docker exec -it docker_mariadb_1 /bin/bash
      (le nom docker_mariadb_1 est celui du container lancé par docker-compose, le vériier avec docker ps),
    • importer manuellement les données dans la base de données :
      mysql -u examen2021 -p examen2021 < examen2021_donnees.sql
    • sortir du container :
      exit
